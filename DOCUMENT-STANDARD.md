# 문서 표준 운영 기준

## 문서 등급

문서 등급: 팀 배포용 표준 운영 가이드

## 운영 환경

운영 환경: 클라우드 기반 테스트 환경

이 문서의 구축, 실행, 검증 기준은 클라우드 기반 테스트 환경을 전제로 한다. 테스트 인프라는 프로젝트별 클라우드 VM, 컨테이너, Kubernetes/NKS 자원으로 구성하고 운영 서비스와 논리적으로 분리한다.

이 저장소의 Markdown 문서는 팀 배포와 반복 운영을 전제로 작성한다. 문서가 단순 메모, 초기 비교 메모, 임시 절차, 실행 미검증 설명으로 남아 있으면 배포 기준을 만족하지 않는다.

## 적용 범위

- 루트 표준 문서
- `docs/` 보고 및 공유 문서
- `Jmeter/`, `K6/`, `nGrinder/`의 `README`, `SETUP-GUIDE`, `TEST-GUIDE`
- k6 런타임 계약 문서
- 운영 하네스 문서
- 커밋 메시지 로컬 운영 규칙

## 필수 문서 기준

- 문서마다 `문서 등급: 팀 배포용 표준 운영 가이드`를 명시한다.
- 문서마다 `운영 환경: 클라우드 기반 테스트 환경`을 명시한다.
- 도구별 `SETUP-GUIDE.md`는 적용 대상, 공식 기준, 사전 조건, 설치 Runbook, 검증 Runbook, 보안/네트워크 기준, 장애 조치, 운영 체크리스트, 구축 완료 기준을 포함한다.
- 도구별 `TEST-GUIDE.md`는 실행 전제, 표준 시나리오 흐름, 실행 Runbook, 성공/실패 판정, 결과 산출물, 장애 조치, 정리/후속 조치를 포함한다.
- 도구별 `README.md`는 역할, 폴더 용도, 표준 템플릿, 먼저 읽을 문서, 운영 포지션, 배포 전 확인을 포함한다.
- 도구별 구축/수행 문서는 클라우드 VM, 컨테이너, Kubernetes/NKS, 전용 node group, 네트워크 분리, 결과 저장 경로 중 해당 도구에 필요한 클라우드 운영 조건을 포함한다.
- 실행 명령은 실제 환경값을 요구해야 하며, 고객 URL, 계정, 토큰, 이벤트 payload를 문서나 템플릿에 하드코딩하지 않는다.
- 버전, 이미지 태그, 설치 경로, 결과 경로는 명시하고 `latest` 태그를 운영 기준으로 사용하지 않는다.
- 검증 상태는 `문서/자산 검증`, `설치 체인 검증`, `실행 체인 검증`, `대상 부하 검증`을 구분해서 적는다.
- 실제 대상 시스템에 부하를 발생시키지 않은 경우에는 설치 또는 dry-run 검증을 대상 부하 검증으로 표현하지 않는다.

## 하네스 강제 기준

`tools/validate-workspace.ps1 -Strict`는 다음을 실패 조건으로 처리한다.

- 필수 문서 또는 실행 자산 누락
- Markdown 상대 링크 깨짐
- 문서 등급 마커 누락
- 클라우드 기반 테스트 환경 마커 누락
- 문서 내 미완료 상태 표기
- 도구별 `README`, `SETUP-GUIDE`, `TEST-GUIDE` 필수 섹션 누락
- 주요 정책 문서에서 JMeter, k6, nGrinder 중 하나라도 누락
- 표준 실행 흐름 `health -> login -> list -> detail -> event` 누락
- 운영 자산의 `latest` 이미지 태그 또는 샘플 계정/샘플 URL 재도입
- k6 정적 ConfigMap 사본 재도입
- 도구별 검증 수준 표기와 실제 수행 범위가 서로 다른 경우

## 배포 판정

팀 배포 전에는 아래 명령이 오류와 경고 없이 통과해야 한다.

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\validate-workspace.ps1 -Strict
```

이 명령이 실패하면 문서 배포를 보류하고, 실패 항목을 먼저 수정한다.
